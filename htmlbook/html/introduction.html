Czym jest HTML?

HTML to skrót od hyper text markup language, czyli hipertekstowy język znaczników, wykorzystywany do tworzenia dokumentów hipertekstowych.

  • HTML opisuje strukturę strony
  • HTML jest językiem znaczników, elementy HTML są reprezentowane przez znaczniki
  • elementy HTML poinformować przeglądarkę w jaki sposób wyświetlać zawartość
  • Przeglądarki nie wyświetlają tagów HTML, ale wykorzystać je do renderowania zawartości strony

Utworzenie pierwszej strony HTML

W celu rozpoczęcia tworzenia kodu HTML nie potrzebujesz połączenia z internetem. Tak naprawdę niezbędna jest jedynie aplikacja pozwalająca na tworzenie plików HTML oraz przeglądarka WWW do ich wyświetlania. Tworzysz, łączysz i testujesz całe zestawy stron internetowych, nie dotykając nawet sieci.

Otwórz edytor tekstu i wprowadź przedstawiony poniżej kod. W tym momencie nie musisz rozumieć dokładnie sposobu jego działania.

<!DOCTYPE html>
<html>
<head>
<title>Moja prosta strona HTML</title>
</head>
<body>
<h1>To jest strona HTML</h1>
</body>
</html>

Wytłumaczenie

  • <!DOCTYPE>Deklaracja określa ten dokument do HTML5
  • <html>Elementem jest element główny od strony HTML
  • <head>Element zawiera meta informacje o dokumencie
  • <title>Element określa tytuł dokumentu
  • <body>Element zawiera widoczną zawartości stron
  • <h1>Element definiuje duży nagłówek
  • <p>Element definiuje paragraf

 Uwaga 

Zwróć uwagę, że znacznik w powyższym przykładzie<!DOCTYPE>, ponieważ jego nazwa rozpoczyna się od wykrzyknika. Celem tego znacznika jest wskazanie weryfikatorom treści i przeglądarkom WWW specyfikacji, zgodnie z którą powstała strona standartu HTML5.
Po utworzeniu pliku HTML zapisz go. Kiedy będziesz wybierał nazwę dla pliku, zastosuj się do dwóch poniższych reguł.
  • Nazwa pliku powinna mieć rozszerzenie .html, na przykład index.html. Jeżeli używasz systemu Windows, upewnij się, że system został skonfigurowany do wyświetlania rozszerzeń plików. Jeżeli nie jest, wówczas skutkiem będzie utworzenie pliku o nazwie takiej jak index.html.txt, który przez przeglądarkę WWW nie będzie rozpoznany jako dukument HTML.
  • Używay krótkich i prostych nazw. Nie stosuj spacji lub znaków specjalnych, a jedynie litery i cyfry. Dzięki temu będziesz mógł łatwo monitorować przeznaczenie poszczególnych plików, a witryna stanie się jeszcze bardziej przyjazna silnikom wyszukiwarek internetowych.

Historia standardów HTML

HTML 2.0 był pierwotnym standardem dla języka HTML i definiował zestaw znaczników, które powinny być obsługiwane przez wszystkie przeglądarki WWW. Większość znaczników określonych w tej pierwotnej specyfikacji nadal jest obsługiwana i stanowi jądro HTML. Dlatego też możesz opracować doskonałe strony internetowe, wykorzystując jedynie znaczniki wprowadzone w specyfikacji HTML 2.0.

Specyfikacja HTML 3.2 została opracowana na początku 1996 roku. Wielu producentów oprogramowania, między innymi Microsoft, IBM, Novell, Netscape Communications Corporation, Spyglass i Sun Microsystems, SoftQuad do konsorcjum W3C, aby pomóc w pracach nad tą specyfikacją. Najważniejsze zmiany wprowadzone w HTML 3.2 obejmowały dodanie funkcji takich jak aplety, teks opływający wokół obrazów i tabele.

Specyfikacja HTML 4.0 pojawiała się w roku 1997 i zawierała wiele nowych funkcji, dzięki którym projektanci zyskali znacznie większą kontrolę nad układem stron niż w przypadku HMTL 2.0 i 3.2 Podobnie jak wymienione wersje HTML, także wersja 4.0 została opracowana przez konsorcjum W3C.

Po specyfikacji HMTL 4.0 pojawiała się XHTML 1.0, a następnie XHTML 1.1. Największą zmianą wprowadzoną w XTHML był wymóg, by dokumenty HTML były prawidłowymi dokumentami XML (ang. extensible markup language, czyli rozszerzalny język znaczników ). Podstawowa różnica między HMLT i XHTML polega na tym, że dokumenty XHMTL muszą być zgodne ze ścisłymi regułami XML dotyczącymi struktury dokumentu. Podczas gdy standard HMTL 4 obsługiwał elementy pozbawione znaczników zamykających, to standard XML wymaga już zamknięcia każdego znacznika. Ponadto każdy atrybut musi mieć wartość itd.

28 października 2014 r. specyfikacja HTML5 osiągnęła status rekomendacji W3C. HTML5 poza dodaniem nowych elementów, usprawniających tworzenie serwisów i aplikacji internetowych, doprecyzowuje wiele niejasności w specyfikacji HTML 4, dotyczących przede wszystkim sposobu obsługi błędów. Niejasności co do sposobu, w jaki przeglądarki powinny obsługiwać błędy w kodzie HTML są jedną z podstawowych przyczyn, dla której wiele serwisów internetowych, napisanych z naruszeniem specyfikacji, w różnych przeglądarkach działa w inny sposób – w niektórych działając, w innych nie.
